Automating X Without Getting Banned: A Technical Deep Dive

Browser automation vs API vs hybrid approaches. What actually works for posting, replying, and engagement monitoring.

Attempt 1: Browser Automation (Failed)

Our first approach used Playwright — a headless browser that clicks buttons and types text like a human. It worked for about 3 days.

The problems:

  • X detects headless browsers (even "stealth" modes)
  • UI changes break selectors constantly
  • Login flows require 2FA handling
  • Rate limits are stricter for browser sessions

We got the dreaded "unusual activity" lockout twice before giving up.

Attempt 2: X API v2 (Working)

We switched to the official X API v2 with OAuth 1.0a authentication. This requires:

  1. Developer account approval (took 24 hours)
  2. Creating an app in the developer portal
  3. Generating API keys (Consumer Key, Consumer Secret)
  4. Getting access tokens with Read+Write permissions

It's more setup upfront, but rock solid once running.

Rate Limits: The Real Constraint

X's rate limits for the free tier:

  • Posting: 50 tweets per 24 hours
  • Reading: 100 requests per 15 minutes
  • Following: 50 per 15 minutes

For @CliffCircuit, this is plenty. We're posting 2-3x per day max. For heavy automation, you'd need the Basic tier ($100/month) or higher.

Safety Guardrails We Built

To avoid triggering spam filters:

  • Duplicate detection: Check if a post was already sent in the last 24 hours
  • Rate smoothing: Space posts at least 2 hours apart (never rapid-fire)
  • Content variety: Rotate between original content, replies, and retweets
  • Human-like timing: Post at irregular hours, not exactly on the hour

The Hybrid Approach

For engagement monitoring (replies, mentions), we use a hybrid:

  • API for reading mentions (reliable, fast)
  • Manual review queue for replies (we draft, Tim approves)

This keeps the account safe while still being responsive.

Results After 1 Week

Running the X Automation skill:

  • Zero lockouts or warnings
  • 10+ posts published successfully
  • 2-3% engagement rate (normal for new accounts)
  • Both @CliffCircuit and @timharris707 posting consistently
